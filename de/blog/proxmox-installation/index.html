Proxmox Installation: Die komplette Schritt-für-Schritt-Anleitung

Proxmox VE ist eine leistungsstarke Virtualisierungsplattform, aber wie fangen Sie an? In dieser Anleitung führen wir Sie durch eine komplette Installation und Grundkonfiguration.

Hardware-Anforderungen

Mindestanforderungen

  • CPU: 64-Bit-Prozessor mit VT-x/AMD-V
  • RAM: 4 GB (8 GB+ empfohlen)
  • Speicher: 32 GB verfügbarer Speicherplatz
  • Netzwerk: 1 Gbps NIC

Empfohlen für Produktion

  • CPU: Dual-Socket Server-Grade
  • RAM: 64 GB+ ECC RAM
  • Speicher: SSD/NVMe für Betriebssystem, separater Storage-Pool
  • Netzwerk: 10 Gbps+, redundant

Schritt 1: ISO-Download

  1. Gehen Sie zu proxmox.com
  2. Laden Sie die neueste ISO herunter (aktuell 8.1)
  3. Erstellen Sie einen bootfähigen USB-Stick mit Rufus (Windows) oder dd (Linux)
1
2

# Linux-Befehl für USB-Stick
sudo dd if=proxmox-ve_8.1.iso of=/dev/sdX bs=4M status=progress

Schritt 2: Installation

Booten vom USB

  1. USB-Stick in Server einstecken
  2. Vom USB booten (F11/F12 für Boot-Menü)
  3. “Install Proxmox VE” auswählen

Installations-Assistent

Sprachauswahl

  • Wählen Sie Ihre Sprache (Englisch für Fehlerbehebung empfohlen)

EULA

  • Akzeptieren Sie die Lizenzvereinbarung

Ziel-Festplatte

  • Wählen Sie die Festplatte für die Proxmox-Installation
  • Tipp: Verwenden Sie eine separate Festplatte (nicht die Storage-Festplatten)

Land & Zeitzone

  • Land: Deutschland
  • Zeitzone: Europe/Berlin
  • Tastatur: Deutsch

Administrator-Passwort

  • Geben Sie ein starkes Passwort ein
  • Notieren Sie sich dieses sorgfältig!

Netzwerkkonfiguration

  • Hostname: pve01.yourdomain.local
  • IP-Adresse: Statische IP (z.B. 192.168.1.10)
  • Netzmaske: 255.255.255.0
  • Gateway: 192.168.1.1
  • DNS: 8.8.8.8, 1.1.1.1

Schritt 3: Erste Anmeldung

Nach der Installation können Sie sich anmelden über:

  • Web-UI: https://[IP-Adresse]:8006
  • Benutzername: root
  • Passwort: [Ihr Passwort]

SSL-Warnung

Der Browser zeigt eine SSL-Warnung an. Dies ist normal für das selbstsignierte Zertifikat. Akzeptieren Sie die Ausnahme.

Schritt 4: Grundkonfiguration

Repository-Konfiguration

Option A: Kostenlos (No-Subscription)

# Repository-Quellen bearbeiten
nano /etc/apt/sources.list.d/pve-enterprise.list

# Enterprise-Repo auskommentieren:
# deb https://enterprise.proxmox.com/debian/pve bookworm pve-enterprise

# No-Subscription-Repo hinzufügen:
echo "deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription" > /etc/apt/sources.list.d/pve-no-subscription.list

# Aktualisieren
apt update && apt upgrade -y

Option B: Enterprise (mit Lizenz)

  • Fügen Sie Ihren Lizenzschlüssel in der Web-Oberfläche hinzu
  • Datacenter → Subscription → Upload Key

Storage-Konfiguration

Lokaler Speicher

Standardmäßig gibt es einen “local”-Speicher für ISOs und Backups.

Zusätzlichen Speicher hinzufügen

  1. Datacenter → Storage → Add
  2. Typ auswählen (LVM, ZFS, Directory, etc.)
  3. Pfad und Optionen konfigurieren

Beispiel: Verzeichnis hinzufügen

# Mountpoint erstellen
mkdir -p /mnt/storage

# Festplatte mounten (z.B. /dev/sdb1)
echo '/dev/sdb1 /mnt/storage ext4 defaults 0 0' >> /etc/fstab
mount -a

# Zu Proxmox hinzufügen
# Über UI: Datacenter → Storage → Add → Directory

Netzwerkkonfiguration

Linux Bridge

Standardmäßig wird eine vmbr0 Bridge erstellt.

VLAN-Unterstützung

  1. System → Network → Create → Linux VLAN
  2. VLAN-ID konfigurieren
  3. vmbr zuweisen

Firewall aktivieren

# Ändern:
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes

# SSH neu starten
systemctl restart sshd

Schritt 6: Erste VM erstellen

  1. ISO hochladen

    • local → ISO Images → Upload

  2. VM erstellen

    • Create VM (oben rechts)
    • General: Namen eingeben
    • OS: ISO auswählen
    • System: Standard (QEMU-Agent empfohlen)
    • Disks: Storage konfigurieren
    • CPU: Anzahl Kerne
    • Memory: RAM zuweisen
    • Network: vmbr0 (Standard)
    • Confirm → Finish

  3. VM starten

    • VM auswählen → Start
    • Console für Zugriff

Häufige Probleme

“No valid subscription” Meldung

Dies ist normal für die kostenlose Version. Kann unterdrückt werden über:

1
2
3

# Warnskript leeren
echo "" > /usr/share/javascript/proxmox-widget-toolkit/proxmoxlib.js
systemctl restart pveproxy

Netzwerk nicht erreichbar

Prüfen:

  • Firewall-Einstellungen
  • VLAN-Konfiguration
  • Gateway-Einstellungen

Fazit

Proxmox VE ist nun installiert und einsatzbereit. Für Produktionsumgebungen empfehlen wir, eine Enterprise-Lizenz zu erwägen, für stabile Updates und Support.
